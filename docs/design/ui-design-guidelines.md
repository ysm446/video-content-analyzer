# UI デザインガイドライン

作成日: 2026-07-03
対象: `frontend/css/common.css`（トークン・基本コンポーネント）/ `frontend/pages/app.html`（画面固有スタイル）

コンセプト: **シンプルかつスタイリッシュ**。
ダーク基調のフラットデザイン。装飾は最小限にし、階層は「背景色の段差＋1px の境界線」だけで表現する。
色はほぼモノトーンに抑え、アクセント（青）は「操作できるもの・進行中のもの」にだけ使う。
他アプリと共有する共通デザインの基準ドキュメントとして書いている。

## 1. カラートークン

`common.css` の `:root` で定義。**色はトークン経由で使い、生の HEX を書かない**（現状の違反は §9 参照）。

| トークン | 値 | 用途 |
|---|---|---|
| `--bg` | `#0d0d0d` | 最背面（ページ背景・動画背景まわり） |
| `--surface` | `#1a1a1a` | パネル・バー・モーダルの面 |
| `--surface2` | `#252525` | 面の中の面（入力欄・リスト項目・ホバー） |
| `--border` | `#2f2f2f` | すべての境界線（1px solid） |
| `--text` | `#e0e0e0` | 本文 |
| `--text-dim` | `#888888` | 補助テキスト・ラベル・非アクティブ |
| `--accent` | `#4a9eff` | 操作可能・進行中・選択中（青） |
| `--accent-hover` | `#6ab3ff` | アクセントのホバー |
| `--success` | `#4caf50` | 完了・ロード済み |
| `--warning` | `#ff9800` | 注意・未保存 |
| `--error` | `#f44336` | 失敗・破壊的操作 |
| `--radius` | `8px` | 基本の角丸 |

### 色の使い方ルール

- **背景の階層は bg → surface → surface2 の3段まで。** それ以上の深さは境界線で区切る
- **アクセントは面積を取らない。** ボタン塗り・ドット・細線・テキストなど小さい要素のみ。
  広い面をアクセントで塗らない（例外はモデルピルのみ＝アプリの「主役」表示）
- **状態色の面表現は「淡い塗り＋やや濃い枠」のティント形式**に統一:
  `background: rgba(色, .08〜.16); border-color: rgba(色, .28〜.4); color: 明るめの同系色`
  （例: 実行中ボタン `rgba(74,158,255,.14)` / 保存ボタン `rgba(255,152,0,.16)`）
- success / warning / error は**意味があるときだけ**使う。装飾に使わない

## 2. タイポグラフィ

- フォント: `'Segoe UI', 'Hiragino Sans', 'Yu Gothic UI', sans-serif`（コードは `ui-monospace, Consolas`）
- ベース 14px / 行間 1.5。**サイズスケールは 11 / 12 / 13 / 14 のみ**を基本にする
  （10px 以下は使わない。字幕・見出しなど表示系のみ例外で 17〜19px）
- ウェイトは 400（本文）と 600（見出し・強調）の2つ。500 はボタンのみ
- **セクション見出しは「小さく・薄く・大文字」**:
  `font-size:11-12px; color:var(--text-dim); text-transform:uppercase; letter-spacing:.06em`
  （`.section-label` / `.modal-section-label` / パネルヘッダで共通）
- 数値（経過秒・時刻・使用率）は `font-variant-numeric: tabular-nums` で桁揃え

## 3. 形状・角丸

| 半径 | 用途 |
|---|---|
| `4px` | 最小要素（サムネール・インラインコード・中止ボタン） |
| `6px` | リスト内の項目・小さな入力欄 |
| `8px`（`--radius`） | 標準（ボタン・カード・入力欄・メニュー） |
| `10–12px` | モーダルパネル・ピル型ボタン |
| `99px / 999px` | 完全な丸（バッジ・ドット・スライダー・円形ボタン） |

境界線は常に `1px solid var(--border)`。太い線・二重線は使わない。

## 4. 余白・寸法

- 基本単位はおよそ **2px グリッド**（gap: 6 / 8 / 10 / 12 / 14、padding: 8〜16）
- 定型の高さ: ステータスバー 28px / ピル・小ボタン 32px / アイコンボタン 34px /
  ヘッダ行 42〜56px
- ボタンのサイズは2種:
  - 標準 `.btn`（padding 10px 20px / 14px）
  - コンパクト（padding 4-7px 10-14px / 12px）… ツールバー・モーダル内はこちら

## 5. エレベーション（影・重なり）

フラット基調。**通常の UI に影は使わない**（面の差＋境界線で表現する）。
影を使うのは「浮いているもの」だけ:

| 要素 | 影 |
|---|---|
| モーダルパネル | `0 24px 64px rgba(0,0,0,.5)` |
| ドロップダウンメニュー | `0 12px 32px rgba(0,0,0,.34)` |
| ツールチップ | `0 6px 24px rgba(0,0,0,.65)` |

モーダルの背景は **暗転＋ぼかし**: `rgba(0,0,0,.45)` + `backdrop-filter: blur(4px)`。

## 6. アイコン

- **Lucide Icons のみ。絵文字は使わない**（`lucide.createIcons()` を動的生成後にも呼ぶ）
- サイズ: 本文中 13–15px / サイドバー 20px / 空状態の飾り 48px
- stroke-width: 1.5–1.9（サイズが小さいほど太め）
- アイコンは必ずテキストかツールチップ（title）を伴う

## 7. モーション

- ホバー・色変化のトランジションは **0.15s**、押下は `transform: scale(.97〜.99)` + 0.1s
- プログレスバーの伸びは 0.3s ease
- ループアニメーションは「進行中」の表現だけに使う:
  - ドット点滅 `blink`（1.2s、opacity 1↔.3）
  - ピルのシマー `pill-shimmer`（1.4s、グラデーション流し）
- バウンス・パララックスなど装飾目的の動きは使わない

## 8. コンポーネント規約

- **ボタン**: `.btn-primary`（アクセント塗り・白字）は1画面に主要操作のみ。
  それ以外は `.btn-ghost`（surface2＋境界線）。破壊的操作は error のティント形式
- **状態ドット**: 6–8px の円。灰（待機）/ 青点滅（実行中）/ 緑（完了）/ 赤（エラー）。
  ステータスバー・ログ・ランタイム一覧で同じ意味づけを守る
- **バッジ**: ピル型（radius 99px）、ティント形式（例: ロード済み＝success 15%）
- **モーダル**: ヘッダ（14px 600・下線）＋本文（padding 14–16px）＋ ×ボタン。
  背景クリック / Esc / × で閉じる。大きい設定系は「左ナビ＋右コンテンツ」の2カラム
- **チャット**: 質問は右寄せの吹き出し（radius 12/12/4/12）、回答は地の文（枠なし）。
  メタ情報（tok/sec 等）はアイコン付きの薄字チップ
- **入力欄**: `surface2` 面＋ `border` 枠。フォーカスは枠線の明色化のみ
  （OS 色のネイティブリングは無効化。派手なフォーカスリングを使わない）
- **スクロールバー**: 幅 6px・トラック透明・つまみは border 色の丸棒

## 9. 改善点と対応状況（2026-07-03 更新）

1. **生 HEX の氾濫** — ✅ 対応済み。`--surface3` / `--border-strong` / `--focus-border` /
   `--accent-dim` / 各状態色のティントトークン（`--*-tint-bg` / `--*-tint-border`）を追加し、
   フォーカス枠・ホバー・シーン項目・ピル・辞書ツールチップ等を置き換えた。
   例外として許容: `#fff` / `#000`（普遍色）、字幕表示色（`#d0d0d0` 等、映像上の可読性優先）
2. **意味色のゆらぎ** — ✅ 対応済み。赤は `--error`、オレンジは `--warning` の各1系統に統一
   （中止ボタン・sysinfo 警告/危険・保存ボタン）
3. **青のゆらぎ** — ✅ 対応済み。モデルピルのグラデは `accent → accent-hover`、
   シマーは `accent-dim ↔ accent-hover`、リンクは `accent-hover` に統一
4. **フォーカス表示の不統一** — ✅ 対応済み。common.css で
   「button は accent の 2px リング（:focus-visible）、入力系は枠明色化のみ」に集約
5. **角丸 7px** — ✅ 対応済み（genre-tag / tag を 6px へ。ピル・モーダルの 10–12px は §3 のとおり許容）
6. **インラインスタイル依存** — ⏸ 未対応。設定モーダルの select・ランタイム行・プロンプト編集 UI を
   `.field-select` `.runtime-row` 等のクラスに抽出して common.css へ移す（次の候補）
7. **common.css の役割強化** — 🔶 一部対応。トークン・フォーカスルール・バッジのティント化は
   common.css に集約済み。モーダル/ドット/入力欄などの汎用コンポーネント移設は 6 と併せて実施する
8. **絵文字の残存** — ✅ 対応済み（チャットのエラー表示 `❌` をテキストに置き換え）
9. **10px 未満の文字** — ✅ 対応済み（elapsed-time / modal-loaded-badge を 11px に）
10. **ライトテーマ非対応** — ⏸ 未対応（1〜3 が済んだので、`data-theme` によるトークン差し替えで
    実現できる状態になった）

残タスクの推奨順: 6・7（構造の分離）→ 10（テーマ切り替え、必要になったら）。
